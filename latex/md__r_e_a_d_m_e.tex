\mbox{\hyperlink{p3_8py}{p3.\+py}} is the card game \char`\"{}\+Dreckschwein\char`\"{} programmed with python. For visualization colorama was used. \hypertarget{md__r_e_a_d_m_e_autotoc_md2}{}\doxysubsection{Our Extension to the game\+:}\label{md__r_e_a_d_m_e_autotoc_md2}
As an extension we decided to add a new card to the deck, the name of the card is {\bfseries{Blitzableiter Gold}}. This card is an altered version of the standard {\bfseries{Blitzableiter}} it has the same function but additionally it allows the player to play another card after using the {\bfseries{Blitzableiter Gold}} effectively allowing the player to play two cards in a turn instead of one. One can not also put this card on {\bfseries{Blitzableiter}}. We can also not put {\bfseries{Blitzableiter}} on {\bfseries{Blitzableiter Gold}} Anzahl von {\bfseries{Blitzableiter Gold}} Karten ist 2.\hypertarget{md__r_e_a_d_m_e_autotoc_md3}{}\doxysection{How users can get started with the project}\label{md__r_e_a_d_m_e_autotoc_md3}
{\bfseries{To have the best playing experience, the user is advised to use a screen resolution of 1920x1080 pixels with scaling set to 100\% in Windows.}}

Input and Output of this application is over the Console. This app can be run by using the command {\bfseries{python \mbox{\hyperlink{p3_8py}{p3.\+py}}}} in console.

Because the game can be played with 2, 3 or 4 players, the user is first asked to input the amount of players that are participating in the match. After setting the desired amount of players, the playing board will be created according to the amount of players.

The game starts with waiting for input of Player\+\_\+0. So each player has a number ranging from \mbox{[}0, \#players -\/ 1\mbox{]}. Each card stack has also a number. All these numbers are visible on the screen. Below the Game panel, the questions will be asked, which control the flow of game by the players. On the right-\/hand side of the panel, there will be a logging panel. Every move by the players will be logged on the logging panel. Also, illegal moves by the players will be shown there. \mbox{\hyperlink{init_8py}{init.\+py}} is the whole backbone of the project it contains the dictionary \char`\"{}game\char`\"{} where the \mbox{\hyperlink{logic_8py}{logic.\+py}} (logic) file takes and writes information into, for the \mbox{\hyperlink{board_8py}{board.\+py}} (UI) file to know what information to display onto the screen. \char`\"{}game\char`\"{} dictionary is also initialized in this module. \char`\"{}game\char`\"{} dictionary is essentially the interface between board and logic module. ~\newline
\hypertarget{md__r_e_a_d_m_e_autotoc_md7}{}\doxysubsubsection{logic.\+py}\label{md__r_e_a_d_m_e_autotoc_md7}
The logic module contains the whole game logic. Every card has its own function that can be called. The functions check if the move is possible or not. ~\newline
 If the move is {\bfseries{possible}} it returns a {\bfseries{True}} value, it then also discards all the cards that need to be discarded and draws a new card from the deck. If the move is {\bfseries{not possible}} it returns a {\bfseries{False}} value and prints the associated error message onto the board log. The function \mbox{\hyperlink{namespacelogic_ab08d9cfe83858b4cd3e334074f690cb0}{step()}} is the main loop, it asks the current player what it wants to do, and the player can simply input his decisions into the console. After every input it is checked if the input was correct, meaning if an integer is asked it also only allows an integer as an answer, else it gives out an error message and asks the user to try again. In combination with the functions of the cards checking if the move is possible, it is ensured that each player can only make legal moves. After a player has done his move, it outputs the made move onto the game log and goes to the next player. After every turn from a player it is checked if there is a winner and if so it breaks out of the loop and returns to the winning player.\hypertarget{md__r_e_a_d_m_e_autotoc_md8}{}\doxysubsubsection{board.\+py}\label{md__r_e_a_d_m_e_autotoc_md8}
